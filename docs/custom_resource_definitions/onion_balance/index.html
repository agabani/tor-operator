Onion Balance

An Onion Balance is an abstraction of a Tor Onion Balance.

Tor Onion Balance is the best way to load balance Tor Onion Services. The load of introduction and rendezvous requests gets distributed across multiple hosts while also increasing resiliency by eliminating single points of failure.

Examples

Basic

The Tor Operator will create an Onion Balance using an auto generated Onion Key load load balancing a list of Onion Services.

#onionkey.yaml
apiVersion: tor.agabani.co.uk/v1
kind: OnionKey
metadata:
  name: onion-balance-{{ include "example.fullname" . }}
  labels:
    {{- include "example.labels" . | nindent 4 }}
spec:
  auto_generate: true
  secret:
    name: onion-balance-{{ include "example.fullname" . }}


#onionbalance.yaml
apiVersion: tor.agabani.co.uk/v1
kind: OnionBalance
metadata:
  name: onion-balance-{{ include "example.fullname" . }}
  labels: {{- include "example.labels" . | nindent 4 }}
spec:
  onion_key:
    name: onion-balance-{{ include "example.fullname" . }}
  onion_services:
    - onion_key:
        hostname: 3evpjb3a5qxaubnzodlxlgn3ccp3achwwwtirgpznnl752oyg2oxekid.onion

Full

The Tor Operator will create an Onion Balance using an auto generated Onion Key load load balancing a list of Onion Services.

#onionkey.yaml
apiVersion: tor.agabani.co.uk/v1
kind: OnionKey
metadata:
  name: onion-balance-{{ include "example.fullname" . }}-full
  labels: {{- include "example.labels" . | nindent 4 }}
spec:
  auto_generate: true
  secret:
    name: onion-balance-{{ include "example.fullname" . }}-full


#onionbalance.yaml
apiVersion: tor.agabani.co.uk/v1
kind: OnionBalance
metadata:
  name: onion-balance-{{ include "example.fullname" . }}-full
  labels:
    {{- include "example.labels" . | nindent 4 }}
spec:
  config_map:
    name: onion-balance-{{ include "example.fullname" . }}-full
  deployment:
    name: onion-balance-{{ include "example.fullname" . }}-full
  onion_key:
    name: onion-balance-{{ include "example.fullname" . }}-full
  onion_services:
    - onion_key:
        hostname: 3evpjb3a5qxaubnzodlxlgn3ccp3achwwwtirgpznnl752oyg2oxekid.onion

Features

State

State can be observed in the status.

kubectl describe onionbalances example

# ...
Status:
  State:     running
# ...

Possible values for State:

  • onion key not found
  • onion key hostname not found
  • running

OpenAPI Spec

apiVersion: apiextensions.k8s.io/v1
kind: CustomResourceDefinition
metadata:
  name: onionbalances.tor.agabani.co.uk
spec:
  group: tor.agabani.co.uk
  names:
    categories: []
    kind: OnionBalance
    plural: onionbalances
    shortNames: []
    singular: onionbalance
  scope: Namespaced
  versions:
  - additionalPrinterColumns:
    - description: The hostname of the onion balance
      jsonPath: .status.hostname
      name: Hostname
      type: string
    - description: The hostname of onion services
      jsonPath: .status.onion_services
      name: Onion Services
      type: number
    - description: Human readable description of state
      jsonPath: .status.state
      name: State
      type: string
    - jsonPath: .metadata.creationTimestamp
      name: Age
      type: date
    name: v1
    schema:
      openAPIV3Schema:
        description: Auto-generated derived type for OnionBalanceSpec via `CustomResource`
        properties:
          spec:
            description: |-
              An Onion Balance is an abstraction of a Tor Onion Balance.

              Tor Onion Balance is the best way to load balance Tor Onion Services. The load of introduction and rendezvous requests gets distributed across multiple hosts while also increasing resiliency by eliminating single points of failure.
            properties:
              config_map:
                description: Config Map settings.
                nullable: true
                properties:
                  name:
                    description: |-
                      Name of the Config Map.

                      Default: name of the Onion Balance
                    nullable: true
                    type: string
                type: object
              deployment:
                description: Deployment settings.
                nullable: true
                properties:
                  name:
                    description: |-
                      Name of the Deployment.

                      Default: name of the Onion Balance
                    nullable: true
                    type: string
                type: object
              onion_key:
                description: Onion Key settings.
                properties:
                  name:
                    description: Name of the Onion Key.
                    type: string
                required:
                - name
                type: object
              onion_services:
                description: Onion Service part of the Onion Balance load balancing.
                items:
                  properties:
                    onion_key:
                      description: Onion Key reference of the Onion Service.
                      properties:
                        hostname:
                          description: |-
                            Hostname value of the Onion Key.

                            Example: "abcdefg.onion"
                          type: string
                      required:
                      - hostname
                      type: object
                  required:
                  - onion_key
                  type: object
                type: array
            required:
            - onion_key
            - onion_services
            title: Onion Balance
            type: object
          status:
            nullable: true
            properties:
              hostname:
                description: |-
                  Onion key hostname.

                  The hostname is only populated once `state` is "running".
                nullable: true
                type: string
              onion_services:
                description: Number of onion services.
                format: int32
                type: integer
              state:
                description: |-
                  Human readable description of state.

                  Possible values:

                  - onion key not found - onion key hostname not found - running
                type: string
            required:
            - onion_services
            - state
            type: object
        required:
        - spec
        title: OnionBalance
        type: object
    served: true
    storage: true
    subresources:
      status: {}
